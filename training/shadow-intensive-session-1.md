## Core Concepts

### OpenClaw Philosophy

**Past (3 years):** AI answering questions, giving advice, writing text  
**Present/Future:** AI executing tasks, building systems, performing steps

> "OpenClaw is actions. It executes tasks, it builds systems, it performs steps."

Key distinction: OpenClaw has:
- **Eyes** — reads screens and data
- **Ears** — understands inputs
- **Hands** — clicks, builds, executes
- **Legs** — moves across systems

### The Opportunity: API Gold Rush

Chad's key insight: Software companies will eventually lock down their public APIs and charge for AI access. Current opportunity is to:

1. Find software with public API documentation
2. Create OpenClaw skills for that software
3. "Bank" those skills before APIs become private/paid

> "We get to run around all night while the rest of the city is sleeping."

### AI Model Strategy

**Three main models:**

1. **ChatGPT** — Good for brainstorming, everyday tasks
2. **Kimi 2.5 (Moonshot)** — 85% cheaper, pleasant experience, good for execution
3. **Claude Sonnet 4.5** — "Big thinker" for creating skills and complex tasks

**Usage pattern:**
- Create skills/commands with Claude (big thinker)
- Execute with Kimi 2.5 (cheap, fast)
- Brainstorm with ChatGPT or Perplexity

## Troubleshooting: The 3 Rs

When OpenClaw has issues:

1. **Reload** — Right-click → Reload (or Ctrl+Shift+R / Cmd+Shift+R)
2. **Restart** — Click restart button in top left
3. **Reset** — Go to Health menu → Reset (takes longer, may lose unsaved work)

> "This will fix 95% of your issues."

**Last resort:** Self-destruct (delete bot and recreate) — but this loses everything.

## Skills Demonstrated

### Memory Bank Skill (Created Live)

Chad created this skill during the workshop in ~2 minutes.

**Purpose:** Upload transcripts/PDFs to create searchable knowledge bases

**Command:** `/memory bank`

**Process:**
1. User drops file (PDF, transcript, etc.)
2. Bot asks: Existing memory bank or new one?
3. If new: User names the bank
4. Bot creates/updates knowledge base
5. User can then ask questions, get summaries, action steps

**Use cases:**
- Workshop transcripts
- Course materials
- YouTube videos
- Webinars
- Any training content

## Key Takeaways

1. **Don't use OpenClaw for brainstorming** — use ChatGPT/Perplexity
2. **Use OpenClaw for execution** — running commands, building systems
3. **Start with cheaper models** (Kimi 2.5) — upgrade to Claude only when needed
4. **Create skills for everything** — document processes as reusable commands
5. **Build memory banks** — feed all training/content into searchable knowledge bases
6. **Train your bot** — it learns from every interaction
7. **Eventually make it CEO** — once trained, let it proactively manage tasks
